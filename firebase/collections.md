# Firestore Collections Schema

## `users` (Employees Only)
- `id` (String, PK - Auth UID)
- `email` (String)
- `name` (String)
- `role` (String: ADMIN, HOST, SECURITY)
- `organizationId` (String)
- `permissions` (Array<String>)

## `visitors` (Permanent Identity)
A Visitor record is created exactly once to prevent duplication.
- `id` (String, PK)
- `name` (String)
- `phone` (String)
- `email` (String)
- `company` (String)
- `governmentId` (String, Hashed/Encrypted)
- `photoUrl` (String, Firebase Storage URL)
- `idCardUrl` (String, Firebase Storage URL)
- `createdAt` (Timestamp)
- `updatedAt` (Timestamp)

## `visits` (Transactional Appointments)
- `id` (String, PK)
- `visitorId` (String, FK -> visitors.id)
- `hostId` (String, FK -> users.id)
- `purpose` (String)
- `expectedEntry` (Timestamp)
- `expectedExit` (Timestamp)
- `actualEntry` (Timestamp)
- `actualExit` (Timestamp)
- `status` (String: PENDING, APPROVED, CHECKED_IN, CHECKED_OUT)
- `approvalStatus` (String)
- `createdBy` (String, FK -> users.id)

## `visitor_passes` (Digital Pass)
Exactly one pass generated per Visit.
- `id` (String, PK)
- `visitId` (String, FK -> visits.id)
- `qrToken` (String, Secure token string)
- `publicUrl` (String, Firebase Hosting URL)
- `status` (String: VALID, EXPIRED, REVOKED)
- `generatedTime` (Timestamp)
- `expiryTime` (Timestamp)
- `viewedTime` (Timestamp)
- `scannedTime` (Timestamp)

## `audit_logs`
- `id` (String, PK)
- `action` (String)
- `userId` (String, FK -> users.id)
- `timestamp` (Timestamp)
- `module` (String)
- `oldValue` (Map)
- `newValue` (Map)
- `device` (String)
